## Patterns/Learnings

- Fork's `packages/agent/src/agent.ts` `prompt()` is a thin overload dispatching to `#runLoop`; upstream's `prompt()` had inline streaming state machine. Many upstream fixes targeting prompt() internals will not apply directly.
- Fork's `packages/ai/src/stream.ts` routes Vertex via a dedicated `streamGoogleVertex` provider function, not via OpenAI-compat fetch wrapper. Upstream's openai-compat + rawPredict path is incompatible.
- Fork's `packages/coding-agent/src/eval/py/` splits into `index.ts` (high-level adapter + `namespaceSessionId`) and `executor.ts` (low-level `executePython`). Upstream consolidated. Agent-session imports `executePython` from `executor.ts` not `index.ts`; need separate re-export for the namespace helper.
- `git cherry-pick -X theirs` insufficient for any upstream commit that imports a module the fork hasn't introduced.
- Baseline test capture uses `bun run check:ts` parallel which truncates per-package output; full visibility requires per-package `cd packages/X && bun test`.
